# 区域图形界面访问 (noVNC)

TheWorld 采用单一区域容器运行时。区域容器默认以图形界面为首要考量（依据 ADR-006 设计决策），并内置可通过浏览器访问的桌面功能。

现已不再提供独立的图形界面容器路径。

## 功能特性

- 每个区域容器内均包含 Linux 桌面环境（XFCE）
- 基于浏览器的远程桌面访问（noVNC 服务运行于容器端口 `3000`）
- 通过 TheWorld 代理路由统一访问：`/gui/:region/*`

## 使用方式

1. 创建区域：

```bash
dio region create -n region-a
```

2. 启动 TheWorld 服务器：

```bash
dio start
```

3. 通过统一代理打开图形界面：

```text
http://127.0.0.1:3344/gui/region-a/
```

4. 可选：查询图形界面状态/端口：

```bash
curl http://127.0.0.1:3344/api/regions/region-a/gui
```

## 安全须知

- 建议默认仅允许本地主机访问（`127.0.0.1`）
- 如需跨机器访问，请使用 VPN 或经过身份验证的反向代理。切勿直接暴露至公共互联网

## 注意事项

- 区域运行时在保持与 `agent` 执行语义兼容的同时，图形界面会话运行于 webtop 的原生桌面用户环境
- 图形界面优先的设计理念更注重功能完整性，而非资源最小化
